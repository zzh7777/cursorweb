# 业务术语与逻辑约定（公共）

本文档为数据库无关的业务知识与判断依据，不依赖 MySQL/PostgreSQL 等具体语法。具体 SQL 实现见各数据库对应文档（如 SKILL.md 的 MySQL 示例）。

## 时间范围的隐含语义

| 规则措辞 | 实际含义 | 实现要点（概念） |
|---------|---------|------------------|
| 无时间限定词 | **就诊级别**（同一次就诊内） | 同一就诊 ID |
| "同时" / "同天" | **同一天内** | 费用发生日期相同 |
| "每日" / "每天" | **按天限额**（跨就诊累计） | 按费用发生日期分组，不限就诊 ID |
| "N小时内" | 指定时间窗口内 | 需计算两时间点的小时差 |
| "N天内不超过M次" | **周期频次限额**（跨就诊、跨机构） | 窗口函数取前一次日期 + 日期差判断间隔，见类型四 |

示例：规则说「A与B同时收费」→ 找同一天内既有 A 又有 B 的记录；规则说「A与B不得重复收费」且无时间词 → 找同一次就诊内既有 A 又有 B；规则说「14天内不超过1次」→ 用窗口函数取上次日期，用日期差判断间隔。

## 违规金额计算（inscp_amt）

明细级违规金额统一用 **符合范围金额** 字段（如 `inscp_amt`），不要用明细费用总额类字段。根据规则类型，计算方式不同：

**类型一：重复/互斥收费**（A与B同时收费，B为违规项）

- A、B 都是集合，不是单个项目
- B 是违规项（后者），违规金额 = B 的符合范围金额直接求和

**类型二：数量超标**（某项目收费次数超过阈值 N，超出部分违规）

- 违规金额 = 总符合范围金额按超出比例分摊
- 公式含义：超出次数占总次数的比例 × 总符合范围金额
- 筛选条件：总数量 > N

**类型三：折扣收费**（A与B共现时，B须按折扣率收费）

- 规则示例：「血气分析同时进行无机元素测定时，无机元素测定须按70%收费」。体现在数据上：B 的数量应为折扣因子的倍数（70% → 数量为 0.7 的倍数）。
- 违规判定：找到不超过实际数量的**最大合理数量**，差值即违规数量。
- 违规金额 = 符合范围金额按违规数量占总数量的比例分摊。
- 通用思路：折扣因子 d = p/q（如 0.7 = 7/10），用整数运算求最大合理数量再求差值，避免浮点精度问题。
- 只对违规数量 > 0 的记录计入。
- 前提：A 与 B 须同天共现（需用子查询或 EXISTS 判断 A 组存在）。

**类型四：周期频次限额**（N天内不超过M次，超出部分违规）

- 规则示例：「平衡试验(视动试验) 14天内支付不超过1次，评定间隔不短于14天」。
- 核心思路：按天聚合后，用窗口函数取前一次收费日期，用日期差算间隔，间隔 < N 天则违规。
- 要点：窗口按参保人分区（跨机构）；违规金额 = 违规次收费的全部符合范围金额（间隔不足则该次不应收费，全额违规）；用日期类型与日期差函数，避免字符串截取日期。
- 若规则为「N天内不超过M次」且 M > 1，窗口函数需向前看 M 行。

## 主动澄清机制

**规则描述存在歧义或信息不足时，必须先向用户提问澄清，确认后再生成 SQL。** 常见需要澄清的情况：

- 规则中的「A」/「B」具体对应哪些项目编码或收费类别
- 阈值的精确数值或计算口径
- 时间范围未明确（哪个月、哪一年）
- 违规金额的计算方式不确定
- 规则适用的机构类型、医疗类别不明确
