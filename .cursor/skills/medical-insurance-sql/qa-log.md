# 问答追踪日志（QA Log）

本文件记录 LLM 在处理医保监管规则时产生的疑问，供人工异步解答，形成知识迭代闭环。

## 使用说明

### 状态流转

```
[PENDING] → 人工填写答案 → [ANSWERED] → LLM 沉淀到知识库 → [INCORPORATED]
```

### LLM 如何提问

在处理规则时遇到不确定因素，按以下模板追加到「问答记录」区域：

```markdown
### Q-XXX [PENDING]
- **来源**: （触发该问题的规则编号或场景描述）
- **提问日期**: YYYY-MM-DD
- **分类**: （业务逻辑 / 数据质量 / 字段含义 / 规则歧义 / SQL 模式）
- **问题**: （清晰描述不确定之处，附上已知信息和推测）
- **影响**: （该问题未解决对 SQL 生成的影响是什么）
```

### 人工如何解答

在对应问题下方补充答案，并将状态改为 `[ANSWERED]`：

```markdown
### Q-XXX [ANSWERED]
- **来源**: ...
- **提问日期**: YYYY-MM-DD
- **分类**: ...
- **问题**: ...
- **影响**: ...
- **答案**: （详细解答）
- **回答日期**: YYYY-MM-DD
- **补充说明**: （可选，附加上下文或参考资料）
```

### 常见提问分类

| 分类 | 典型场景 |
|------|---------|
| 业务逻辑 | 某规则的违规判定标准不明确，如"重复收费"的时间窗口如何界定 |
| 数据质量 | 发现某字段空值率异常高，不确定是否可用 |
| 字段含义 | 某字段的业务含义不确定，或多个字段间的关系不明 |
| 规则歧义 | 规则文本存在多种理解方式，需确认正确解读 |
| SQL 模式 | 不确定某种场景下的最优 SQL 写法 |

### Q-002 [INCORPORATED]
- **来源**: 规则「同一患者同天重复收取血常规和血沉检测费用」
- **提问日期**: 2026-03-02
- **分类**: 规则歧义
- **问题**: "重复收取"的含义是什么？存在两种理解：(1) 血常规与血沉互斥，同天同时收取即违规（因为血常规已包含血沉检测）；(2) 同一项目在同天被重复计费（如血常规收了两次）。两种理解对应完全不同的 SQL 逻辑。
- **影响**: 理解 (1) 用 EXISTS 交叉判定两组项目共现，理解 (2) 用 COUNT/SUM 判定单项目重复。SQL 结构完全不同。
- **答案**: 指血常规与血沉互斥，血常规已包含血沉检测，同天同时收取时血沉为违规项
- **回答日期**: 2026-03-02

### Q-003 [INCORPORATED]
