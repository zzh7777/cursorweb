# 自审检查（公共）

SQL 写完后，逐项过一遍。以下为概念性检查项，具体字段名、函数以当前数据库文档为准。

- [ ] 所有表名、字段名符合当前库规范（如小写）
- [ ] 字符串字段与字符串比较，数值字段与数值比较，类型匹配
- [ ] 日期时间字段用日期函数处理；字符串型日期需按库要求处理
- [ ] 住院天数用起止日期差计算，不用「住院天数」字段（若空值率高）
- [ ] 「同天」用日期部分比较或分组，不用字符串截取
- [ ] 医疗类别、医院等级等在同名字段不同表中的类型一致（整型/字符型）
- [ ] 已过滤有效标志（如 vali_flag = 1）
- [ ] 退费记录处理正确（通常保留对冲）
- [ ] 若涉及结算表与明细表 JOIN，确认金额字段未因一对多而膨胀
- [ ] GROUP BY 包含所有非聚合 SELECT 列
- [ ] 结果包含参保人标识 + 机构标识 + 违规金额
- [ ] 时间范围边界与语义正确：无限定词=就诊级，「同时」=同天，「N小时内」=时间差，「N天内M次」=窗口函数+日期差
- [ ] 周期频次规则：按参保人分区（跨机构），用日期类型与日期差，不用字符串截取
- [ ] 违规金额用符合范围金额字段，且计算方式与规则类型匹配（重复收费→直接求和，数量超标→按比例分摊，折扣收费→最大合理数量差值法，周期频次→全额违规）
- [ ] 折扣收费规则：用整数运算避免浮点精度（如 0.7 写为 7/10 参与运算）
- [ ] 已排除不适用的特殊记录（按需）
- [ ] 规则有歧义时已先向用户澄清，未自行假设
